Are API parameter values and calls case-sensitive?
â–¼
Yes, all API parameter values and calls are CASE SENSITIVE. Exact spelling and capitalization must be used. This is especially important for operator names in UMM queries, which must match exactly as shown in the GUI dropdown. For example, 'indicator=Firm Booked' works, but 'indicator=Firm booked' returns no results. Users should copy parameter values directly from referential endpoints to avoid mismatches.

â–¼
Yes, the API calls are in JSON format by default.

â–¼

How do I filter API results by multiple parameter values?
â–¼
Use comma-separated values for a single parameter. For example: indicator=Nomination,Allocation returns data for Nomination OR Allocation. Multiple parameters are separated by '&'.
How do I filter UMMs to show only active events?
â–¼

How do I get only the latest version of a UMM?
â–¼

How do I include Not Applicable (N/A) data in API results?
â–¼
Use the includeExemptions parameter. Set includeExemptions=1 to return populated data plus N/A instances (exemptions under Regulation 715). Use isNA=1 to return only N/A records, or isNA=0 to return only valid data.

â–¼
In the API call, use 'from' and 'to' parameters (not 'periodFrom' and 'periodTo'). Format: YYYY-MM-DD. The platform converts these to the appropriate periodFrom/periodTo based on the indicator's periodicity (hourly/daily).

What are best practices for optimizing API calls?
â–¼
Avoid concurrent requests, add cool down time if receiving HTTP Code 429, always use pointDirection parameter in aggregatedData calls, limit date ranges to avoid timeouts, and schedule automated calls during off-peak hours (18:00-06:00 CET).
What are the API execution limits and why should I use date filters?
â–¼
The Transparency Platform limits API execution to 60 seconds per call, and all APIs return a maximum of 100 rows by default (override with limit parameter). Date period limits are essential to ensure queries complete within the timeout and to make requests more efficient. Users should apply filters (date ranges, points, operators, indicators) to avoid timeouts.
When should I send API requests and what affects performance?
â–¼
The optimal time for API requests is 18:00-06:00 Central European time (lowest server load). Peak usage is 6 AM - 12 PM CET. API performance can be affected by high server load during peak hours or platform maintenance. The platform implements throttling (max 6 calls per minute) to balance load. Check for published UMMs about ongoing issues if problems persist outside peak hours.

What are Urgent Market Messages (UMMs) and are they mandatory on the Transparency Platform?
â–¼
UMMs are voluntary publications based on the REMIT Regulation, which governs inside information disclosure in energy markets. While it is mandatory to publish inside information, it is not mandatory to publish it on the Transparency Platform itself. Nevertheless, around 20 of the 45 TSOs currently use the TP for publishing their UMMs.

â–¼
The Transparency Platform provides links to: booking platforms (e.g., PRISMA, GSA in Poland, RBP in Southeast Europe), storage facility information via AGSI+, and LNG terminal information via ALSI. For each relevant point, you can access the corresponding booking platform link directly from the TP interface.
What happened to the Favorites feature on the ENTSOG Transparency Platform?
â–¼
The previous Favorites feature, where users could create smaller groups of points within their profile, was discontinued after the platform introduced a personalized login feature. However, users can still achieve the same functionality by using API calls.
What happens during platform maintenance periods?
â–¼
During maintenance periods, operational data processing may be temporarily interrupted. ENTSOG publishes Urgent Market Messages (UMMs) to inform users of any issues. Maintenance periods can affect both data availability on the platform and API call responses.

â–¼
When a TSO is replaced (for example, when Ukrtransgaz was replaced by Gas TSO LLC of Ukraine), the platform is updated to reflect the new operator. Historical data from the previous TSO remains available, and new data is published under the new TSO name.

â–¼

What were the transparency requirements for GTS before ENTSOG-TP?

When and how are unplanned interruptions published?
â–¼
Unplanned interruptions are defined as events taking place less than 42 days in the future. TSOs publish them both as Urgent Market Messages (UMMs) and in the interruptions view on the platform. When did GTS start publishing transparency data?
â–¼
GTS started publication of numerical data before Regulation 715/2009, initially with definitions and formats defined by GTS. The publication evolved through bilateral agreements with the EC, national requirements by NRA, and eventually Regulation 715/2009 requirements before moving to ENTSOG-TP.
When does the 'last updated' date change for published data?
â–¼
The 'last updated' date only changes if there's a new value sent to the platform. If the TSO continues to send the same information repeatedly, the last updated date will not be updated to be more effective in the data warehouse. This means that if data hasn't changed for a long time, it doesn't necessarily mean the TSO hasn't sent data.
When is data updated on the Transparency Platform?
â–¼
Data update schedules vary by TSO and data type. Each Transmission System Operator publishes data according to their own schedule and regulatory requirements. There is no single platform-wide update schedule. For specific data types or points, the timing depends on when the relevant TSO makes the data available.

For TSOs in hourly balancing regimes, nominations, renominations, and physical flows must be sent on an hourly basis. GCVs and Wobbe index are optional hourly indicators. How is unlimited interruptible capacity displayed on the Transparency Platform?
â–¼
When TSOs publish unlimited interruptible capacity (meaning you can book as much as you want with no limit), it appears as 'unlimited' in the transport data table and as zero in the graph view, since unlimited capacity cannot be displayed numerically in a chart.
What is the difference between compact and periodized data format?
â–¼
Compact format shows data for variable periods - one record might cover days, months, or even a year if values don't change (e.g., 239 entries for 10 years). Periodized format displays data per day, so the same period would have many more entries (e.g., 30 entries for one month instead of 1). Periodization affects capacity and interruption indicators.
What are item remarks and why are they important?
â–¼
Item remarks are additional notes that can be published with any indicator value. For example, they might indicate conditional products like 'B setK' (a German firm conditional product). Remarks contain valuable information about how the data correlates with products on booking platforms, and they are also available in downloads.
What do the colored circles (red and green) indicate in the Transport Data table?
â–¼
A red circle next to a record indicates provisional data. When the TSO sends confirmed data, the circle turns green. This is particularly important for physical flows and gas qualities because TSOs don't always have measurements available as quickly as they need to publish the data, so the data might still be subject to change if it's provisional.
What does 'not applicable' (N/A) mean in downloaded data?
â–¼
Not applicable (N/A) appears when an exemption applies. For example, for a Virtual Reverse Flow point where physical flow can only go in one direction, the other direction shows as N/A because flow is physically impossible in that direction. Hovering over N/A or checking the item remark provides more information about the exemption reason.
Who is responsible for the accuracy of data on the Transparency Platform?
â–¼
Each Transmission System Operator (TSO) is responsible for the accuracy and publication of their own data. ENTSOG cannot comment on or modify TSO data publications as this is within each TSO's own remit. Questions about specific data accuracy should be directed to the relevant TSO.
Why am I seeing duplicate or multiple flow values for the same interconnection point?
â–¼
Multiple flow values can appear when different operators report data for the same point. ENTSOG cannot determine which flow is the correct one as this is the responsibility of the TSOs. If you encounter this issue, ENTSOG can forward your question to the relevant TSOs for clarification.
Why is data missing for certain points or time periods?
â–¼
Missing data can occur for several reasons: maintenance periods at the specific point, TSO publication delays, or the point may not be required to publish certain data types. If you notice unexpected missing data, you can contact ENTSOG support for clarification about specific points.
How can I break down gas consumption by sector using the platform?
â–¼
You can use keypoint codes to filter consumption data by sector. However, availability varies by country and depends on whether TSOs are required to publish distribution-level data. Not all member states publish comprehensive sector-level consumption data due to regulatory requirements.
How does the platform handle nomination versus renomination data in exports?
â–¼
Nominations and renominations are grouped together. If there is no renomination, the nomination value will show. If both nomination and renomination exist, the renomination will be displayed as it represents the most current information.
Should I use Physical Flow or Renomination data for actual gas transfers?
â–¼
For actual gas transferred between countries, use Physical Flow data. Physical Flow represents the actual measured gas flow, while Renomination represents scheduled or nominated flows that may differ from actual physical transfers.
What are the different types of interruptions published on the Transparency Platform?
â–¼
There are firm interruptions (planned and unplanned) and interruptible interruptions (planned interruption of interruptible capacity or actual interruption of interruptible capacity). Planned interruptions must be published by November 15th for the following year. Unplanned interruptions are events taking place less than 42 days in the future.
What are the six capacity indicators available on the Transparency Platform?
â–¼
There are three firm capacity indicators: Firm Technical Capacity, Firm Booked Capacity, and Firm Available Capacity. There are three interruptible capacity indicators: Total Interruptible Capacity, Booked Interruptible Capacity, and Available Interruptible Capacity.
What is an EIC code?
â–¼
EIC (Energy Identification Code) is a unique code for a point or an operator in the European energy business. While EIC codes don't change often, they can theoretically change, which is why point keys and TSO keys are also provided as more stable identifiers.
What is an OPD (Operator Point Direction) on the Transparency Platform?
â–¼
OPD stands for Operator Point Direction. Points are always shown based on the combination of operator, point name, and direction. This combination is called an OPD or sometimes just referred to as 'the point', but it's important to remember it's always these three elements together.
Why are point keys and TSO keys useful in exported data?
â–¼
Point keys and TSO keys are stable unique identifiers. Even if an operator changes their name or a point name changes, the keys remain the same. This makes them very useful for tracking the same operator or point over time, especially when using the API. EIC codes may also change, but keys provide consistent identification.
What is the Unplanned Interruptions section and what does it show?
â–¼
The Unplanned Interruptions section shows only unplanned firm interruptions and actual interruptible interruptions (not planned interruptions). Information is displayed in tabular format showing point, operator, direction, and event start/stop dates. You can include or exclude zero values, and the data is exportable.

What are the main aims of EU Network Codes?
â–¼
Network Codes aim to achieve six objectives: enhancing transparency, removing entry barriers, increasing short-term liquidity, creating better connected markets, increasing competition, and harmonizing legislation across the EU.
What do CAM and CMP Network Codes achieve?
â–¼
CAM (Capacity Allocation Mechanisms) and CMP (Congestion Management Procedures) achieve harmonized auctions and bundled capacity products while avoiding contractual congestion.
What does BAL NC promote?

BAL NC (Balancing Network Code) promotes short-term markets and market-based harmonized balancing.
What is TAR NC and what are its benefits?
â–¼
TAR NC (Tariff Network Code) develops cost-reflective and non-discriminatory tariffs to minimize cross-subsidies and ensures transparency in tariff methodologies.
What is the FUNC Coordination Group?
â–¼
The FUNC Coordination Group is a coordinated mechanism between ENTSOG, ACER, and the European Commission that validates, categorizes, and prioritizes issues related to Network Codes and Guidelines, then develops and approves proposed solutions.
What is the FUNC process?
â–¼
FUNC (Functionality process) is a coordinated mechanism of ACER and ENTSOG, supported by the EC, to develop and propose issue solutions for matters relating to Network Codes and Guidelines. The process collects issues via the FUNC platform, validates, categorizes and prioritizes them, then develops, approves and publishes proposed solutions.
When was the BAL NC implemented?

The BAL NC was implemented on 1 October 2015, with a possibility for later implementation until 1 October 2016.
When was the Incremental Capacity Process implemented?
â–¼
The Incremental Capacity Process was implemented on 6 April 2017 under the CAM NC.
When were VIPs implemented under CAM NC?
â–¼
Virtual Interconnection Points (VIPs) were implemented on 1 November 2018 under the CAM NC.

What does the 'P&P' notation mean on the Transparency Platform?
â–¼
P&P stands for 'pipe-in-pipe', which means multiple TSOs are active on the same infrastructure in the same direction. When you hover over the indicator, you can see which TSOs are sharing the same infrastructure. You can hide individual TSO legends to better see overlapping data.
What is Virtual Reverse Flow?
â–¼
Virtual Reverse Flow means that a physical point only has one actual flow direction, but capacity can be purchased in both directions. One direction is virtual, and it's up to the TSO to coordinate with the TSO on the other side to ensure capacity delivery.
What is a pipe-in-pipe configuration on the Transparency Platform?
â–¼
Pipe-in-pipe means that a single physical pipeline is shared by several TSOs. Each TSO manages its own capacity, though the physical gas flow is common. In such cases, you may see extra data rows (additional capacity or flow information), explanatory notes from operators, and up to six tabs representing different flow directions or interconnections.
What is a virtual point on the ENTSOG Transparency Platform?
â–¼
A virtual point is indicated by a different-shaped balloon and includes 'virtual' in its name. It shows the virtual point itself and the underlying physical points that are aggregated within it. Virtual points may have multiple tabs showing flows in different directions, and physical points included in a virtual point are indicated by a small arrow pointing into the virtual point.

Can you access booking platforms directly from Point Information?
â–¼
Yes, Point Information includes links to booking platforms that you can click to be taken directly to the relevant booking platform for that point.
What does the 'last update date' mean in Point Information?
â–¼
For master data in Point Information, the last update date shows when the database was last checked for new information. It doesn't necessarily mean something changed; it just confirms the information is up to date.
What information is displayed in the Point Information section?
â–¼
Point Information shows: operator name, EIC code of the operator (not the point), direction, CAM relevance status, whether CMP measures are applied, Virtual Reverse Flow indicator, validity period of the point, average GCV value (if provided), unit and conversion factor, last update date, and Pipe-in-Pipe indicator.
What is the Pipe-in-Pipe indicator in Point Information?
â–¼
The Pipe-in-Pipe indicator is the newest column in Point Information. It shows when multiple TSOs share the same physical infrastructure. This indicator was already available on the map and in point pop-ups, and was added to Point Information for consistency and easy reference in the overview.
What is the purpose of the Point Information section on the Transparency Platform?
â–¼
Point Information contains static master data maintained by TSOs. It's useful for checking naming conventions since different markets and TSOs may use different names for the same point. For example, a point might be called 'AN2' in a search but have different names used by different operators like Open Grid Europe or Thyssengas.
What product information is shown in Point Information?
â–¼
Point Information shows which capacity products are offered by each TSO for the point (yearly, monthly, daily, etc.). Products marked in green are currently offered. Some TSOs may still show multi-annual or half-annual products (not aligned with CAM network code) because they have old contracts where these products remain relevant, even if not actively offered.
Why are unit and conversion factor still shown in Point Information?
â–¼
Unit and conversion factor are considered legacy information from a time when multiple units were available on the platform. Now everything is provided in kWh. While there's debate about removing these fields, they've been kept because deleting them might result in losing some information that could still be useful.
Why is the validity period of a point important in Point Information?
â–¼
The validity period is especially relevant when you cannot find certain information. For example, you might be looking at a planned point where only capacity information is published before the point is commissioned. For planned points, there would be no physical flows or gas quality data since they are not yet operational.
Why might average GCV values be missing from Point Information?
â–¼
Since GCV (Gross Calorific Value) data is now published every day (or hourly) on the platform, some TSOs have removed the average GCV value from Point Information because it's not as accurate as the real-time information they provide hourly or daily.
Tariffs17
How are discounts displayed in the tariff data?
â–¼
Discounts are shown in the additional parameters block. For example, FGSZ provides a 10% discount for interruptible yearly products, shown as a decimal value. Some TSOs like Gas Connect Austria apply ex-post discounts, with the formula and additional remarks shown in the data block.
How are tariff values displayed in different currencies on the ENTSOG Transparency Platform?
â–¼
TSOs provide tariff data in their local currency, and the conversion into Euro is done automatically by the Transparency Platform using the European Central Bank's exchange rate. The platform displays both the original currency value and the Euro conversion.
How does the Transparency Platform handle tariffs published in non-Euro currencies?
â–¼
When TSOs publish tariffs in their local currency (e.g., British pounds), the platform automatically recalculates to Euros using the European Central Bank exchange rate. The platform shows both the applicable tariff in the common unit (Euros) and in the local currency, along with the exchange rate reference date used for the conversion.
How frequently is tariff data updated?
â–¼
Tariff data is updated at irregular intervals as TSOs publish new information. There is no fixed schedule for tariff updates. Check the API regularly or monitor the lastUpdateDateTime field to track when new tariff information becomes available.
Is tariff information available for all points on the Transparency Platform?
â–¼
No, the Transparency Platform only has tariff information for border points (interconnection points). For internal points or other TSO points, you should look at the TSO's own website or the NRA's website, where more comprehensive tariff information is available.
What additional tariff information is available when expanding the details?
â–¼
When you expand the additional information (clicking the plus signs), you can see: the multiplier, discount, seasonal factor, the applicable value in local currency (not just Euros), connection information, balancing zone, currency, last update date, exchange rate reference date, general remarks, and commodity charges if applicable.
What are simulation costs in tariff data?
â–¼
Simulation costs represent the cost for transporting 1 GWh per day for a year via a particular product type (yearly, quarterly, monthly, daily, within-day). They must be published one month before the tariff period and allow users to compare costs across different product types. Typically, yearly products are cheapest, with shorter-term products being progressively more expensive.
What are the two types of tariff information published on the Transparency Platform?
â–¼
The two types are: 1) Tariff data (reserve prices), which must be published one month before the yearly auction, and 2) Commodity charges, which must be published one month before the tariff period. Simulation costs are also published, showing the cost for transporting 1 GWh/day for a year via particular products.
What does 'N/A' mean in tariff data and what information is provided?
â–¼
N/A stands for 'not applicable' and indicates an exemption. When you hover over N/A, you can see the exemption code/reason, the start time and end time of validity for the exemption during that tariff period, and additional information about the exemption.
What does the tariff period represent and can it vary by country?
â–¼
The tariff period is the time period for which tariffs are valid. Each member state has its own tariff period - some use calendar years, others run from October to October, April to April, or other annual periods. The tariff period can differ for each zone.
What is the 'applicable tariff in common unit' and what units does it use?
â–¼
The 'applicable tariff in common unit' is the standardized tariff value that allows comparison across TSOs. For products that are within a day or longer, it's always expressed in Euro per kWh per day. For within-day products, it's expressed in Euro per kWh per hour. This value already includes the multiplier, seasonal factor, and interruptible discount.
What is the Simulation Data section used for on the ENTSOG Transparency Platform?
â–¼
The Simulation Data section shows values provided for simulation costs. By hovering over tooltips, you can read additional remarks such as the tariff period and simulation cost explanation, including which parameters were considered by the TSO when providing the value. Note that simulation costs are displayed only for firm yearly products.
What is the common unit used for tariff comparability across European TSOs?
â–¼
The 'Applicable Tariff in Common Unit' ensures comparability among all TSOs across Europe and is displayed as: kWh per day for most products, and kWh per hour per hour for within-day products.

CMP Data7
What are the four types of CMP (Congestion Management Procedures) information available on the Transparency Platform?
â–¼
The four types are: 1) Unsuccessful requests - when a shipper's capacity request cannot be fully met at auction, 2) Auction results - when capacity is sold above reserve price, 3) Unavailable firm capacity - when expected capacity is not offered, and 4) Capacity made available - when additional capacity is released through surrender, over-subscription, or UIOLI (Use It Or Lose It) short-term or long-term.
What are the four ways capacity can be made available under CMP?
â–¼
Capacity can be made available through: 1) Surrender of capacity, 2) Over-subscription, 3) UIOLI (Use It Or Lose It) short-term, and 4) UIOLI long-term. Each method has a separate record showing how much capacity was made available through that specific CMP measure.
What does an auction result in CMP data indicate?
â–¼
An auction result is published when, for products of one month or longer, capacity was sold at a price higher than the reserve price at auction. This typically occurs when demand exceeds supply. The data shows the auction period, auction premium (extra amount paid), clearing price (final sale price), and reserve price (original offered price).
What does an empty record in CMP data mean?
â–¼
Empty records in CMP data indicate that everything was loaded and there was no case to report for that operator point direction. For example, if there's only an empty record for unsuccessful requests, it means there were no unsuccessful requests for that point during that period.
What does it mean when TSOs publish zero values for capacity made available?
â–¼
When a TSO publishes zero values for capacity made available, it means they did not make any capacity available through that CMP measure during that period. TSOs are not required to publish zero values (publishing nothing means the same thing), but some choose to do so for transparency.
What is an unsuccessful request in CMP data?
â–¼
An unsuccessful request occurs when, for monthly products or longer at an auction, a shipper's request cannot be fully met. For example, if a customer wants 100 kWh/h but only 50 kWh/h can be offered, the request is unsuccessful. This information is published showing the requested amount versus the amount actually given.
What is unavailable firm capacity in CMP data?
â–¼
Unavailable firm capacity information is published for products of one month or longer when capacity that normally should have been offered at auction was not offered. For example, if monthly capacities are usually offered at auction but are suddenly not offered for one auction, this is recorded as unavailable firm capacity.
UMMs6
What are the two types of Urgent Market Messages (UMMs) on the Transparency Platform?
â–¼
The two types are: 1) Unavailability of gas facilities - used for situations like reduced capacity due to maintenance, and 2) Other market information - often used for IT issues (e.g., nomination system downtime), capacity unavailabilities that are uncertain, or when you don't know the precise end date yet.
What does the 'active' status mean for Urgent Market Messages?
â–¼
Active status mostly means UMMs that apply at the moment. However, it's important to check the event start and stop dates because a UMM might be marked as active even if it's completely in the past (the status may not have been updated to inactive). Always verify the dates to confirm current applicability.
What information is displayed in the Unavailability of Gas Facilities UMM section?
â–¼
The section shows: who published the information, which balancing zone, event start and stop dates, status (inactive/active/dismissed), event type, affected plant, reason for unavailability, affected point direction, unit, original capacity, still available capacity, unavailable capacity, remarks, UMM ID (unique identifier), version number, and whether it's the latest version.
What is the UMM ID and version tracking system?
â–¼
The UMM ID is a unique identifier for each specific UMM. You can have several versions of the same UMM, and the 'Latest Version' field shows whether you're looking at the most current version. You can click to show all versions, with the latest version displayed on top.
What is the difference between unavailability of gas facilities UMMs and other market information UMMs?
â–¼
Unavailability of gas facilities UMMs contain detailed capacity and technical information (affected points, capacity values, plant information). Other market information UMMs have less information, typically just a remark explaining what's happening (e.g., 'IT nomination system issue'). They serve different communication purposes.
When did the current UMM publication format become aligned with ACER requirements?
â–¼
The platform adjusted its UMM publication format in July 2018 to align with ACER requirements. UMMs published before July 25, 2018 were under a previous solution and are still available on the platform for transparency, though they may no longer be active.
